# Лето Андрея: 99 ночей в лесу

Веб-приложение для летнего трекера Андрея: ночи, миссии, чтение, умения, открытия, сложные дела, помощь по дому, награды и календарь.

## Главная схема

Есть две разные части: код приложения и данные Андрея.

Код приложения:

`Codex -> папка на компьютере -> GitHub Desktop -> GitHub online -> Replit -> опубликованный сайт`

Данные Андрея:

`локальный HTML / опубликованный сайт -> Supabase`

## Где лежит код

Рабочая папка на компьютере:

`/Users/svetlanasoloveva/Documents/New Project 3`

Codex правит файлы именно в этой папке.

Основные файлы:

- `index.html` — входная страница сайта для Replit/GitHub.

- `andrey-summer-adventure.html`

Этот файл нужен только для открытия на компьютере. В GitHub его отправлять не надо.

## Как обновляется код

1. В Codex меняем приложение.
2. Файлы меняются в папке `/Users/svetlanasoloveva/Documents/New Project 3`.
3. GitHub Desktop показывает изменённые файлы.
4. В GitHub Desktop делаем `Commit to main`.
5. Потом нажимаем `Push origin` или `Publish branch`.
6. Изменения появляются в онлайн GitHub.
7. Replit берёт код из онлайн GitHub и обновляет опубликованный сайт.

## Где хранятся реальные данные

Реальные данные не хранятся в GitHub.

Они сохраняются в общей детской Supabase:

- книги;
- календарь;
- события;
- миссии;
- награды;
- родительские настройки;
- текущий прогресс;
- статусы дней.

Таблица Supabase:

`app_state_documents`

Ключ семьи:

`andrey-family`

Сейчас приложение Андрея подключено к той же детской Supabase, что и приложение Кирилла:

`https://sgiqymstsmgvjighdelz.supabase.co`

Старая Supabase Андрея не удаляется и остается резервной копией.

## Перенос в общую детскую Supabase

Чтобы данные Андрея появились в общей базе:

1. Открыть Supabase с адресом `https://sgiqymstsmgvjighdelz.supabase.co`.
2. Открыть `SQL Editor`.
3. Выполнить весь SQL из файла `supabase_schema.sql`.
4. После success открыть сайт Андрея и обновить страницу.

SQL можно запускать повторно: запись `andrey-family` аккуратно обновится тем же сохраненным снимком.

## Локальная версия и опубликованный сайт

Локальный HTML и опубликованный сайт подключены к одному Supabase.

Можно отмечать прогресс и локально, и на опубликованном сайте, но лучше не делать это одновременно.

Безопасное правило:

1. Внесли изменения в одной версии.
2. Подождали пару секунд, чтобы Supabase сохранил данные.
3. Если открываете другую версию, сначала обновите страницу.

Иначе последнее сохранение может перезаписать более раннее, потому что приложение пока сохраняет весь снимок состояния целиком.

## Что считается кодом, а что данными

Код:

- новые кнопки;
- новый раздел;
- изменение дизайна;
- изменение правил начисления;
- изменение логики календаря;
- изменение родительского режима.

Это делается через Codex, потом commit/push в GitHub.

Данные:

- добавить книгу;
- отметить миссию;
- добавить событие в календарь;
- начислить награду;
- добавить помощь по дому;
- отметить ночь как пройденную.

Это делается прямо на сайте и сохраняется в Supabase.
